# Baseline (before) screenshots — DEFERRED

Stage 0 of plan 09 calls for "before" screenshots of the old UI across several
states, themes, and font scales. **These are deferred to manual on-device
capture by the project owner**, because no Android emulator is available in the
build environment used to implement this plan.

The following states should be captured here when a device/emulator is
available:

- Main screen with no setup complete
- Partially complete setup
- All required setup complete
- Server dialog
- Current update controls and representative result Toasts (record the absence
  of visible in-progress/result state as a baseline defect — fixed in Stage 5)
- Blocked screen with and without allowed sites
- Light and dark mode

For each capture, record the device/API, window size, theme, and font scale.
